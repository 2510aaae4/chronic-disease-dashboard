# 「臺灣 50」優良 SMART on FHIR 應用程式徵案 - 報名指南

## App Demo URL 說明

在「電子病歷應用程式 內容說明（限1000字數）」欄位中，主辦單位要求提供 **App Demo URL**。

### 這是什麼？

一個可以讓評審實際展示/試用應用程式的公開網址。

### 為什麼需要？

評審需要實際操作你的 SMART on FHIR 應用程式，而不僅是閱讀文字描述。

---

## SMART on FHIR 啟動流程

```
評審操作流程：

1. 評審登入 Taiwan Sandbox (thas.mohw.gov.tw)
        ↓
2. 輸入 App Demo URL: https://xxx.github.io/chronic-disease-dashboard/launch.html
        ↓
3. 選擇測試病患 → 點「啟動」
        ↓
4. Sandbox 呼叫 launch.html，帶上參數：
   https://xxx.github.io/chronic-disease-dashboard/launch.html?iss=...&launch=...
        ↓
5. launch.html 執行 FHIR.oauth2.authorize()
        ↓
6. 重導向到 index.html，顯示病患資料
```

### 重點說明

| 問題 | 答案 |
|------|------|
| 評審需要你的帳號密碼嗎？ | ❌ 不需要 |
| 評審用誰的登入？ | 用他們自己的 Sandbox 帳號 |
| 病患資料從哪來？ | 從 Sandbox 的 FHIR Server |
| 你需要提供什麼？ | 只需要公開的 launch.html URL |

### 概念說明

`launch.html` 就像一個**門**：
- 你把門的地址給評審 (App Demo URL)
- 評審用他們的**鑰匙** (Sandbox 帳號) 開門
- 進去後看到的**房間內容** (病患資料) 是 Sandbox 提供的

## 部署方式

由於 CareDash 是純靜態 HTML/JS 應用程式，可使用以下方式部署：

| 部署平台 | 範例 URL |
|---------|---------|
| GitHub Pages | `https://你的帳號.github.io/chronic-disease-dashboard/launch.html` |
| Vercel | `https://caredash.vercel.app/launch.html` |
| Netlify | `https://caredash.netlify.app/launch.html` |

**建議使用 GitHub Pages**（免費、簡單）

## CareDash 展示方式

### 方式 1: launch.html (標準 SMART Launch)

- **URL**: `https://xxx.github.io/chronic-disease-dashboard/launch.html`
- **用途**: 評審透過 Sandbox 啟動，完整 OAuth 流程
- **優點**: 展示完整的 SMART on FHIR 整合能力

### 方式 2: demo.html (Demo 模式)

- **URL**: `https://xxx.github.io/chronic-disease-dashboard/demo.html`
- **用途**: 直接開啟即可看到預設病患資料
- **優點**: 不需要 Sandbox，任何人都能直接查看

### 建議

在內容說明中同時提供兩個 URL：

```
App Demo URL:
- SMART Launch: https://xxx.github.io/chronic-disease-dashboard/launch.html
- 獨立展示模式: https://xxx.github.io/chronic-disease-dashboard/demo.html
```

## GitHub Pages 部署步驟

1. 建立 GitHub Repository（如尚未建立）
2. 將 `chronic-disease-dashboard` 資料夾內容推送到 repo
3. 進入 Repository Settings → Pages
4. Source 選擇 `main` branch，資料夾選 `/ (root)`
5. 儲存後等待部署完成
6. 取得 URL: `https://你的帳號.github.io/repo名稱/launch.html`
